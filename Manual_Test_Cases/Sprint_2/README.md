# Sprint 2 – Test Design (EP & BVA) and Requirement Analysis

## Overview
This sprint focused on requirement analysis, test planning interpretation, and structured test case design using Equivalence Partitioning (EP) and Boundary Value Analysis (BVA) for the Urban Routes application.

The main focus was validating input field behavior in accordance with functional requirements FR-1 to FR-4, with detailed test design applied to FR-4.

---

## What I did

### Requirement Analysis
- Decomposed FR-1 to FR-4 into atomic testable blocks
- Identified functional behavior for UI elements, input handling, and validation rules
- Ensured requirements were broken into clear and measurable conditions

### Test Planning Review
- Reviewed provided test plan structure
- Identified relevant sections for test case alignment and execution tracking

### Test Design (FR-4 Focus)
- Created equivalence classes for valid and invalid input scenarios
- Defined boundary values for input length constraints (50-character limit)
- Designed negative test cases for invalid characters and edge cases
- Selected test data for each equivalence class and boundary condition

### Test Execution
- Executed test cases based on designed scenarios
- Recorded results in spreadsheet format
- Verified system behavior against expected outcomes

---

## Key Testing Techniques Used
- Equivalence Partitioning (EP)
- Boundary Value Analysis (BVA)
- Negative testing
- Input validation testing
