# Sprint 1 – Test Execution Summary

## Objective
Execute pre-defined functional test cases for the Urban Routes map-based application.

---

## Scope of Testing
The following functional areas were tested:

- Map interactions (scrolling and zooming)
- UI element behavior (unclickable map regions)
- Input field functionality ("From" and "To" fields)
- Search and location selection behavior
- Rendering of map objects (3D objects, landmarks, metro stations, parks)
- Fullscreen mode functionality
- Map rendering modes (landscape and satellite)
- Object information display
- Street view functionality
- Application information modal

---

## Execution Notes
- All provided test cases (CASE-1 to CASE-15) were executed manually in a controlled test environment.
- Each test case was validated against its expected result as defined in the test documentation.
- UI behavior and map rendering responses were observed during execution.

---

## Results Summary
- Majority of test cases behaved as expected based on functional requirements.
- UI interactions such as map navigation, input field selection, and rendering modes functioned correctly under test conditions.
- No structured defect reporting was performed for this sprint.

---

## Outcome
The application’s core map functionality was validated through execution of CASE-1 to CASE-15 test cases, covering navigation, input handling, rendering, and UI behavior.
