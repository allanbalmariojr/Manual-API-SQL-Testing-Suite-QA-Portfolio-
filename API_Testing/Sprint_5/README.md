# Sprint 5 – SQL Data Analysis & Validation

## Overview
This sprint focused on SQL-based data analysis and validation of a backend database. The goal was to extract, filter, and aggregate data using SQL queries to support business and analytics requirements.

## Scope
The following areas of the database were tested and analyzed:

- Startup company status (open vs closed)
- Funding and investment data across countries and sectors
- Acquisition trends (cash-based deals)
- Influencer identification using pattern matching
- Venture fund activity classification and behavior analysis
- Funding round volatility and statistical distribution

## Activities
- Analyzed database structure and relationships
- Wrote SQL queries using SELECT, WHERE, GROUP BY, HAVING, CASE, and aggregate functions
- Applied filtering, sorting, and pattern matching techniques
- Performed data aggregation and business logic validation
- Verified outputs against expected analytical requirements

## SQL Concepts Used
- SELECT statements
- WHERE clause filtering
- LIKE pattern matching
- GROUP BY and HAVING
- Aggregate functions (COUNT, SUM, AVG)
- CASE statements
- ORDER BY sorting

## Artifacts
- SQL query file (`sql_queries.sql`)
- Requirement analysis document
- Query output validation (manual verification)

## Outcome
All business questions were translated into SQL queries successfully. The results demonstrated the ability to extract meaningful insights from structured data, including funding trends, investor behavior, and market activity patterns.

This sprint reinforced core SQL skills in data filtering, aggregation, and analytical querying.
